## 目的

- `web/admin`は現在素のHTML・JavaScriptのパーツでコンポーネントを作成しているため、実装コストが高い。`web/apps`で導入済みのMantine v8をadminにも導入して、使いやすいUIコンポーネントを低コストで導入・保守できるようにしていきたい。
- その他、ツギハギで作ってしまった各編集ページのUIを刷新して、素早くミスがない編集が可能なようにしていきたい。

## 導入後の展望

- 現在adminから編集不可な以下のリソースのページを作成
	- lines全体
	- operators全体
	- stationsのよみがな以外も含めた基本情報すべて
- 未解決路線にGeminiの無料APIを利用し、キーとなるODPTのIDから日本語名と路線コードを推測して入力できるようにする。
- DBとの通信している様子がわかるように、以下を導入する。
	- 登録中の待機表示
	- ページ読み込み時のスケルトンUI
	- ドロップダウンやコンボボックスの選択肢を動的に取得する際のローディングスピナー
- UIを日本語に統一する
- スマートフォンにも対応したレスポンシブデザインを導入する
